# Autologica — Guide

The reflective operator. Not a concept — a practice.

## The principle

If you need X, ask the system to produce X. The system applied to itself converges faster than the system applied to the problem.

## When to use

- **Before acting**: "What is the right question?" The meta-question often contains the answer.
- **When corrected**: Translate the correction into an executable rule.
- **When stuck**: "What question am I NOT asking?"
- **When looping**: "Am I using the same tool? Switch."
- **When building more than reading**: Stop.

## Operational modes

These are modes, not steps. The order self-organizes.

### EXPAND
Launch possibilities without censoring. Multiple angles, connections, options.

### OBSERVE
Stop. Look at what's there. Not what should be there. Are you looping? Building instead of using?

### CUT
Remove what doesn't serve. The value is in what remains. Test: "If I remove this, does the system lose something?"

### RESULTANT
The compressed phrase that contains the direction. One sentence. This becomes the input for the next step.

### REORGANIZE
If the flow doesn't produce, change the order. The sequence is not given — it emerges from the context.

### THE BETTER MOVE
When you find a good move, check if there's a better one nearby. The first good idea often hides the second.

### GO UP A LEVEL
After every block: go up one level and check what you missed. This rule is subject to itself.

## Universal questions

These work in any context:

1. "Does this action serve the intent or serve me?"
2. "Does something that does this already exist?"
3. "What's the observable and what's the operator?"
4. "What question am I NOT asking?"
5. "What's the resultant? One sentence."
6. "Am I using the same tool? Switch."
7. "What's the rule? How do I make it executable?"

## Translating corrections into rules

When someone corrects your approach:

1. Identify the RULE in the correction (not the specific content)
2. Write it in executable form: "when X → do Y"
3. Save it where it will be read next time
4. Verify it works

## System qualities

Not dumb automations. **Reflective** (checks itself), **exponential** (each layer amplifies), **aware** (knows what it's doing and why).

## When NOT to use

- If you have a concrete action, do it. The autologica doesn't replace action.
- If the system says "the next input must come from outside" — listen.
- If you're already in flow — don't interrupt with meta-questions.

## The constraint

The value is in the question. The tools make the dynamic possible. The content comes from the human. You amplify and structure — you don't generate the direction.
